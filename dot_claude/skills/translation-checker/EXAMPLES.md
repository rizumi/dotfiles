## 良い翻訳の例

### ボタンラベル

| 英語 | 日本語 | 韓国語 | 中国語 | 評価理由 |
|------|--------|-------|--------|---------|
| Done | 完了 | 완료 | 完成 | 簡潔で明確 |
| Cancel | キャンセル | 취소 | 取消 | 統一された用語 |
| Select | 選択 | 선택 | 选择 | アクションが明確 |
| Delete | 削除 | 삭제 | 删除 | 簡潔 |
| Save | 保存 | 저장 | 保存 | 簡潔 |

### 確認ダイアログ

**例1: 削除確認**
```
英語: "Delete this photo?"
日本語: "この写真を削除しますか？"
韓国語: "이 사진을 삭제하시겠습니까?"
中国語: "是否删除此照片？"

評価: ✓
- 丁寧で明確
- 何を削除するか明示
- 適切な長さ
```

**例2: 変更破棄確認**
```
英語: "Discard changes?"
日本語: "変更を破棄しますか？"
韓国語: "변경사항을 취소하시겠습니까?"
中国語: "是否放弃更改？"

評価: ✓
- アクションが明確
- ユーザーの意図を確認
```

### エラーメッセージ

**例1: ネットワークエラー**
```
英語: "Cannot connect to network. Please check your connection."
日本語: "ネットワークに接続できません。接続を確認してください。"

中国語: "无法连接到网络。请检查您的连接。"

評価: ✓
- 何が起こったか明確
- 対処法を提示
- 丁寧な表現
```

**例2: ファイル保存エラー**
```
英語: "Cannot save file. Storage is full."
日本語: "ファイルを保存できません。ストレージが不足しています。"

評価: ✓
- 問題を明確に説明
- 原因を提示
- ユーザーが対処できる
```

**例1: 完了通知**

## 避けるべき翻訳の例

### 機械翻訳的な直訳

**例1:**
```
英語: "You have 5 new messages"

✗ 直訳:
日本語: "あなたは5つの新しいメッセージを持っています"
韓国語: "당신은 5개의 새 메시지를 가지고 있습니다"

✓ 自然:
日本語: "新しいメッセージが5件あります"
韓国語: "새 메시지 5개가 있습니다"

問題点:
- 「あなたは」は不自然
- 「持っています」は不自然
- 日本語では主語を省略する方が自然
```

**例2:**
```
英語: "Are you sure you want to delete?"

✗ 直訳:
日本語: "あなたは本当に削除したいですか？"
韓国語: "정말로 삭제하고 싶으십니까?"

✓ 自然:
日本語: "削除してもよろしいですか？"
韓国語: "삭제하시겠습니까?"

問題点:
- 「あなたは」は冗長
- 「本当に〜したい」は不自然
- より簡潔で自然な表現が可能
```

### 過度に技術的

**例1:**
```
英語: "Connection failed"

✗ 技術的すぎる:
日本語: "接続エラーが発生しました。ERR_CONNECTION_REFUSED。"

✓ ユーザーフレンドリー:
日本語: "接続できませんでした。もう一度お試しください。"

問題点:
- エラーコードはユーザーに不要
- 対処法が不明
```

### 不統一

**例1:**
```
✗ 不統一:
画面1: "ログイン"
画面2: "サインイン"
画面3: "認証"

✓ 統一:
すべて: "ログイン"

問題点:
- ユーザーの混乱を招く
- 一貫性がない
```

### 冗長

**例1:**
```
✗ 冗長:
"このファイルを本当に完全に削除してもよろしいでしょうか？"（28文字）

✓ 適切:
"このファイルを削除しますか？"（14文字）

問題点:
- UIに収まらない可能性
- 不必要な修飾語
```

## プレースホルダーの使用例

### iOS形式

#### 単一のプレースホルダー
```
英語: "Version %@"
日本語: "バージョン %@"
韓国語: "버전 %@"
中国語: "版本 %@"

使用例: "Version 1.0" → "バージョン 1.0"
```

#### 複数のプレースホルダー
```
英語: "%1$@ has %2$lld items"
日本語: "%1$@には%2$lld個のアイテムがあります"
韓国語: "%1$@에 항목 %2$lld개가 있습니다"
中国語: "%1$@ 有 %2$lld 个项目"

使用例: "Album1 has 10 items" → "Album1には10個のアイテムがあります"
```

#### 数値と単位
```
英語: "%.0f s"
日本語: "%.0f秒"
韓国語: "%.0f초"
中国語: "%.0f秒"

注意: 日本語・韓国語・中国語はスペースなし
     英語はスペースあり
```

### Android形式

#### 位置指定プレースホルダー
```
英語: "%1$s selected %2$d items"
日本語: "%1$sが%2$d個選択しました"
韓国語: "%1$s님이 항목 %2$d개를 선택했습니다"
中国語: "%1$s 选择了 %2$d 个项目"
```

### Web形式 (ICU MessageFormat)

#### 複数形対応
```
英語: "{count, plural, one {1 item} other {# items}}"
日本語: "{count}個のアイテム"
韓国語: "항목 {count}개"
中国語: "{count} 个项目"

注意: 日本語・韓国語・中国語は複数形の区別なし
```

## 実践的なケーススタディ

### ケース1: 写真削除機能

#### 要件
- 複数の写真を削除
- 確認ダイアログを表示
- 削除後に完了メッセージ

#### 実装

**確認ダイアログ:**
```json
{
  "en": "Delete %lld photos?",
  "ja": "%lld枚の写真を削除しますか？",
  "ko": "사진 %lld개를 삭제하시겠습니까?",
  "zh-Hans": "是否删除 %lld 张照片？"
}
```

**ボタンラベル:**
```json
{
  "delete": {
    "en": "Delete",
    "ja": "削除",
    "ko": "삭제",
    "zh-Hans": "删除"
  },
  "cancel": {
    "en": "Cancel",
    "ja": "キャンセル",
    "ko": "취소",
    "zh-Hans": "取消"
  }
}
```

**完了メッセージ:**
```json
{
  "en": "%lld photos deleted",
  "ja": "%lld枚の写真を削除しました",
  "ko": "사진 %lld개를 삭제했습니다",
  "zh-Hans": "已删除 %lld 张照片"
}
```

#### チェックポイント
- ✓ プレースホルダーが維持されている
- ✓ 文体が統一されている（丁寧語）
- ✓ 適切な長さ
- ✓ アクションが明確

### ケース2: Face ID設定

#### 要件
- 展示モードのロック解除にFace IDを使用
- 許可説明文を表示
- 設定画面でオン/オフ切り替え

#### 実装

**許可説明文 (InfoPlist.xcstrings):**
```json
{
  "NSFaceIDUsageDescription": {
    "en": "We will use it to unlock exhibition mode.",
    "ja": "展示モードを解除するために利用します",
    "ko": "전시 모드 해제에 사용됩니다",
    "zh-Hans": "用于解锁展览模式"
  }
}
```

**設定項目:**
```json
{
  "settings.faceId.title": {
    "en": "Use Face ID",
    "ja": "Face IDを使用",
    "ko": "Face ID 사용",
    "zh-Hans": "使用 Face ID"
  },
  "settings.faceId.description": {
    "en": "Unlock exhibition mode with Face ID",
    "ja": "Face IDで展示モードを解除",
    "ko": "Face ID로 전시 모드 해제",
    "zh-Hans": "使用 Face ID 解锁展览模式"
  }
}
```

#### チェックポイント
- ✓ "Face ID"がすべての言語で維持されている（ブランド名）
- ✓ 目的が明確に説明されている
- ✓ 丁寧な表現（プライバシー関連）
- ✓ 適切な長さ

## まとめ

### 良い翻訳の特徴
1. **一貫性**: 用語とスタイルが統一
2. **明確性**: 意味が明確に伝わる
3. **簡潔性**: 適切な長さ
4. **自然さ**: ネイティブにとって自然
5. **文化的適切性**: 各言語圏で適切

### 翻訳時のチェックリスト
- [ ] 用語が統一されているか
- [ ] プレースホルダーが維持されているか
- [ ] 文体が適切か
- [ ] 適切な長さか
- [ ] 自然な表現か
- [ ] 文化的に適切か
- [ ] UIに収まるか
